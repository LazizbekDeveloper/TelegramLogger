## Обзор возможностей

### Пересылка событий в реальном времени
- **Вход/выход игроков** — Мгновенные уведомления с количеством онлайн игроков
- **Обнаружение первого входа** — Специальные приветственные сообщения для новых игроков
- **Мониторинг чата** — Весь игровой чат пересылается в Telegram с HTML форматированием
- **Отслеживание смертей** — Подробные сообщения о смерти с информацией об убийце
- **Система достижений** — Уведомления о достижениях игроков (открытия рецептов фильтруются)
- **Мониторинг миров** — Отслеживание перемещений между измерениями (Обычный мир, Незер, Энд, пользовательские миры)
- **Логирование команд** — Мониторинг выполнения команд с настраиваемой фильтрацией
- **Запуск/остановка сервера** — Уведомления при запуске или остановке сервера

### Интеграция с Telegram
- **Двусторонний чат** — Отправка сообщений из Telegram напрямую в Minecraft
- **Поддержка тем форума** — Организация сообщений в темах форума Telegram
- **Удалённые команды сервера** — Выполнение любой серверной команды через `/sudo` с полным захватом вывода
- **Статус сервера** — Проверка статуса сервера, TPS, памяти и онлайн игроков из Telegram
- **Система администраторов** — Регистрация администраторов Telegram с постоянным хранением
- **HTML форматирование** — Богатое форматирование сообщений с жирным, курсивом, блоками кода и цитатами

### Защита и надёжность
- **Защита от флуда** — Ограничение скорости предотвращает затопление сообщениями в обоих направлениях
- **Автовосстановление конфигурации** — Повреждённые конфигурационные файлы резервируются и автоматически пересоздаются
- **Фильтрация чата** — Блокировка нежелательных слов с уведомлениями администраторов
- **Чёрный список Sudo** — Предотвращение удалённого выполнения опасных команд
- **Предотвращение дублирования сообщений** — Защита опроса устраняет дублирование сообщений

### Возможности для разработчиков
- **Модульная архитектура** — Чистая структура пакетов с разделением ответственности
- **Режим отладки** — Подробное логирование для устранения неполадок
- **Отслеживание статистики** — Всеобъемлющий подсчёт сообщений и событий
- **Автопроверка обновлений** — Уведомляет администраторов сервера о доступности новой версии

## Что нового в v5.0.0

### Исправления ошибок
- **Исправлены дублирующиеся сообщения Telegram** — Сообщения из Telegram иногда доставлялись в Minecraft 2-3 раза. Причиной были перекрывающиеся запросы long-poll. Исправлено с помощью атомарной защиты опроса.
- **Исправлены сбои команды `/sudo`** — Команда sudo вылетала в группах без форума, поскольку пыталась получить доступ к `message_thread_id` даже при его отсутствии. Теперь корректно обрабатывает отсутствующие ID тем.
- **Исправлено отсутствие вывода `/sudo`** — Ранее `/sudo plugins` показывал "выполнено успешно", но не показывал вывод. Теперь захватывает и отображает полный вывод команды в Telegram.
- **Исправлен спам префикса многострочных сообщений** — Многострочные сообщения из Telegram имели префикс плагина перед каждой строкой. Теперь префикс применяется только один раз.
- **Исправлено повреждение HTML-сущностей** — HTML-сущности вроде `&lt;` повреждались при очистке цветовых кодов, превращаясь в `t;`. Теперь HTML-сущности корректно сохраняются на протяжении всего конвейера сообщений.
- **Исправлен флуд сообщений Telegram при `/reload`** — Использование `/reload` из Telegram вызывало бесконечный цикл, так как смещение обновлений терялось при перезагрузке. Теперь перезагрузка доступна только из игры.
- **Исправлен сбой `/sudo` с ванильными командами** — Пользовательский подход CommandSender вызывал сбои с ванильными командами Minecraft. Теперь используется консольный отправитель с захватом логгера для надёжного вывода.
- **Исправлен мусор от плагинов упоминаний в Telegram** — Плагины упоминаний вставляют в сообщения теги компонентов вроде `<chat=UUID:text>`. Теперь они очищаются перед отправкой в Telegram.

### Новые возможности
- **Уведомления о запуске/остановке сервера** — Получайте уведомления в Telegram при запуске или остановке сервера
- **Обнаружение первого входа** — Специальные приветственные сообщения для игроков, входящих впервые
- **Система защиты от флуда** — Настраиваемое ограничение скорости предотвращает затопление сообщениями
- **Захват вывода Sudo** — Команды `/sudo` теперь показывают полный вывод в ответе Telegram
- **Чёрный список Sudo** — Блокировка опасных команд (`stop`, `op`, `ban-ip` и др.) от удалённого выполнения
- **Команда TPS** — Проверка TPS сервера и использования памяти из Telegram командой `/tps`
- **Автовосстановление конфигурации** — При повреждении конфигурации плагин автоматически создаёт резервную копию, генерирует новую конфигурацию и мигрирует восстанавливаемые значения
- **Замена префиксов** — Замена некрасивых игровых префиксов рангов на чистый текст для сообщений Telegram через конфигурацию `prefix_replacements`
- **Плейсхолдеры префикса/суффикса** — Новые плейсхолдеры `%prefix%` и `%suffix%` извлекают теги рангов из отображаемых имён игроков
- **Очистка компонентов чата** — Автоматическое удаление тегов компонентов, вставленных плагинами, из сообщений чата перед пересылкой

### Улучшения кода
- **Полная реструктуризация кода** — Монолитный файл на 3000 строк разделён на 11 специализированных классов в 6 пакетах
- **Чистая архитектура** — Разделение ответственности: конфигурация, API Telegram, обработка событий, команды, управление данными
- **Улучшенная обработка ошибок** — Понятные сообщения об ошибках вместо молчаливых сбоев
- **Экранирование HTML** — Пользовательский ввод правильно экранируется для предотвращения HTML-инъекций в сообщениях Telegram

## Руководство по установке

### Шаг 1: Скачивание и установка

1. Скачайте последний `TelegramLogger-5.0.1.jar` из [Releases](https://github.com/LazizbekDeveloper/TelegramLogger/releases)
2. Поместите `.jar` файл в папку `plugins/` вашего сервера
3. Запустите сервер один раз для генерации конфигурации по умолчанию
4. Остановите сервер для редактирования конфигурации

```
plugins/
  TelegramLogger-5.0.1.jar
  TelegramLogger/
    config.yml          <-- Редактируйте этот файл
    data.json           <-- Автоматически генерируемая статистика
    admins.json         <-- Автоматически генерируемый реестр администраторов
```

### Шаг 2: Создание Telegram бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/botfather)
2. Отправьте `/newbot` и следуйте инструкциям:
   ```
   /newbot
   > Введите имя бота: Мой Сервер Бот
   > Введите username бота: my_server_bot
   ```
3. Скопируйте **API Токен**, который даст BotFather (выглядит как `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`)

4. **Важные настройки бота** (отправьте BotFather):
   ```
   /setprivacy    -> Выберите бота -> Disable
   /setjoingroups -> Выберите бота -> Enable
   ```

5. (Необязательно) Установите команды бота для удобного меню:
   ```
   /setcommands -> Выберите бота -> Вставьте это:
   status - Показать статус сервера
   stats - Статистика сообщений
   players - Онлайн игроки
   start - Начать пересылку
   stop - Остановить пересылку
   debug - Переключить режим отладки
   tps - Производительность сервера
   sudo - Выполнить серверную команду
   help - Показать команды
   ```

### Шаг 3: Получение Chat ID

1. Создайте группу Telegram (или используйте существующую)
2. Добавьте бота в группу **как администратора**
3. Временно добавьте [@RawDataBot](https://t.me/rawdatabot) в группу
4. Найдите `"chat":{"id": -123456789}` в ответе — это число ваш **Chat ID**
5. Удалите RawDataBot из группы

**Для групп-форумов (с темами):**
- Отправьте сообщение в конкретную тему, куда хотите получать уведомления
- RawDataBot покажет `"message_thread_id"` — это ваш **Thread ID**

### Шаг 4: Настройка плагина

Отредактируйте `plugins/TelegramLogger/config.yml`:

```yaml
# Обязательно: Токен бота от BotFather
bot_token: "123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"

# Обязательно: Chat ID вашей группы/канала
chat_id: "-1001234567890"

# Необязательно: Thread ID для тем форума
thread_id: "123"
send_to_thread: false

# Включить двусторонний чат (сообщения Telegram появляются в Minecraft)
send_telegram_messages_to_game: true
```

Сохраните файл и выполните `/tl reload` в игре или перезапустите сервер.

**Зарегистрируйте себя как администратора** (необходимо для команд Telegram):
```
/tl admin add ВАШ_TELEGRAM_USER_ID ВашеИмя
```
Чтобы узнать свой Telegram user ID, отправьте сообщение в группу с RawDataBot и найдите `"from":{"id": 123456}`.

## Справочник конфигурации

### Настройки бота

```yaml
bot_token: "BOT_TOKEN"                    # Токен API вашего Telegram бота
chat_id: "CHAT_ID"                        # ID целевой группы/канала
thread_id: "THREAD_ID"                    # ID потока темы форума
send_to_thread: false                     # Включить поддержку тем форума
send_telegram_messages_to_game: false     # Пересылать сообщения Telegram в Minecraft
```

### Префикс сообщений

```yaml
plugin_prefix: "&6&lTelegramLogger&7 ➜ &r&a"   # Игровой префикс сообщений (поддерживает цветовые коды)
telegram_game_message: "&7[&9TG&7] &c%name% &8» &f%message%"   # Формат сообщений TG→MC
```

Шаблон `telegram_game_message` определяет, как сообщения Telegram отображаются в Minecraft. Доступные плейсхолдеры: `%name%` (имя отправителя), `%message%` (текст сообщения).

### Замена префиксов

```yaml
prefix_replacements:
  "VIP": "[VIP]"
  "ADMIN": "[ADMIN]"
  "Member": "<tg-emoji emoji-id='5368324170671202286'>👤</tg-emoji>"
```

Замена некрасивых игровых префиксов рангов на чистый текст для сообщений Telegram. Ключ — текст для поиска в отображаемом имени игрока, значение — на что заменить. Префикс получается через Vault API (работает с LuckPerms, PEX и др.) с откатом к извлечению из отображаемого имени, если Vault не установлен.

**Поддержка HTML:** Значения замены поддерживают полный Telegram HTML, включая пользовательские эмодзи (`<tg-emoji>`), жирный, курсив и другие теги. Значения отправляются как необработанный HTML без экранирования. Если ни одна замена не совпала, префикс автоматически экранируется для безопасности.

### Уведомления о запуске/остановке сервера

```yaml
enable_server_start_stop: true
server_start_message: "<blockquote>🟢 <b>Сервер запущен!</b>\nВерсия: %version% | Макс. игроков: %max%</blockquote>"
server_stop_message: "<blockquote>🔴 <b>Сервер остановлен!</b></blockquote>"
```

Отправляет уведомление в Telegram при запуске или остановке сервера. Плейсхолдеры: `%version%`, `%max%`.

### Сообщения о входе/выходе

```yaml
enable_join: true
join_message: "<blockquote>➕ <b>%player%</b> зашёл в игру! (%online%/%max%)</blockquote>"

enable_leave: true
leave_message: "<blockquote>➖ <b>%player%</b> вышел из игры! (%online%/%max%)</blockquote>"
```

### Первый вход (Новый игрок)

```yaml
enable_first_join: true
first_join_message: "<blockquote>🌟 <b>%player%</b> зашёл впервые! Добро пожаловать! (%online%/%max%)</blockquote>"
```

Срабатывает только когда игрок заходит на сервер впервые (определяется через `player.hasPlayedBefore()`).

### Сообщения чата

```yaml
enable_chat: true
chat_message: "💬 <b>%player%</b> ➥ %message%"
```

Все игровые сообщения чата пересылаются в Telegram. Пользовательские сообщения экранируются для предотвращения HTML-инъекций.

### Сообщения о достижениях

```yaml
enable_advancement: true
advancement_message: "<blockquote>🏆 <b>%player%</b> получил <b>[%advancement%]</b> (%online%/%max%)</blockquote>"
```

Достижения по открытию рецептов автоматически фильтруются.

### Сообщения о смерти

```yaml
enable_death: true
death_message: "<blockquote>💀 %death_message% (%online%/%max%)</blockquote>"
```

Включает полное сообщение о смерти Minecraft с информацией об убийце.

### Сообщения о смене мира

```yaml
enable_world_switch: true
world_switch_message: "<blockquote>🌍 <b>%player%</b> перешёл: %from_world% → %to_world% (%online%/%max%)</blockquote>"
```

Названия миров автоматически форматируются с эмодзи:
| Название мира | Отображение |
|---|---|
| `world` | 🌍 Overworld |
| `world_nether` | 🔥 The Nether |
| `world_the_end` | 🌌 The End |
| `spawn` | 🏔️ Spawn |
| `lobby` | 🏔️ Lobby |
| `custom_world` | 🌎 Custom World |

### Фильтр чата

```yaml
enable_chat_filter: true
filtered_words:
  - "плохоеслово1"
  - "плохоеслово2"
  - "оскорбление"
filtered_message: "<blockquote>🚫 <b>%player%</b> использовал запрещённое слово. (%online%/%max%)</blockquote>"
```

При обнаружении запрещённого слова:
1. Оригинальное сообщение **не** пересылается в Telegram
2. Вместо него отправляется уведомление о фильтрации
3. Игровые администраторы получают предупреждение

### Логирование выполнения команд

```yaml
enable_send_command_executes: false
command_execute_message: "<blockquote>💠 <b>%player%</b> ➥ <code>%command%</code> (%online%/%max%)</blockquote>"
command_executes_chat_id: "CHAT_ID"           # Может отличаться от основного чата
send_command_executes_to_thread: false
command_executes_group_thread_id: "THREAD_ID"
ignored_commands:
  - "/login"
  - "/register"
  - "/l"
  - "/reg"
```

Команды из списка `ignored_commands` не будут логироваться. Можно использовать отдельный чат/тему Telegram для логов команд, чтобы основной чат оставался чистым.

### Telegram Sudo команда

```yaml
enable_sudo_command: false
sudo_show_output: true
sudo_blacklist:
  - "stop"
  - "restart"
  - "op"
  - "deop"
  - "ban-ip"
```

При включении зарегистрированные администраторы могут выполнять серверные команды из Telegram:

```
/sudo list          → Показывает онлайн игроков
/sudo plugins       → Показывает установленные плагины с выводом
/sudo kill Steve    → Убивает игрока Steve
/sudo say Привет!   → Отправляет сообщение всем
/sudo gamemode creative Steve → Меняет игровой режим
```

Ответ показывает:
- Выполнена ли команда успешно
- Какая команда была выполнена
- Какой администратор выполнил
- Полный вывод команды (если `sudo_show_output: true`)

Команды из `sudo_blacklist` блокируются для безопасности.

### Защита от флуда

```yaml
anti_flood_enabled: true
anti_flood_max_messages: 20     # Максимум сообщений в окне
anti_flood_window_seconds: 10   # Длительность окна в секундах
```

Предотвращает затопление в ситуациях:
- Массовый вход/выход игроков
- Спам в чате
- Быстрое получение достижений
- Шторм событий

Сообщения, превышающие лимит скорости, молча отбрасываются.

### Режим отладки

```yaml
debug_mode: false
```

Включите через `/tl debug` в игре или `/debug` из Telegram. Показывает подробные логи для:
- Обработки сообщений
- Вызовов API Telegram
- Загрузки конфигурации
- Решений по ограничению скорости
- Проверок администраторов

## Управление администраторами

Администраторы — это пользователи Telegram, которые могут:
- Отправлять сообщения из Telegram в игровой чат Minecraft
- Использовать команды Telegram (/status, /stats и др.)
- Выполнять серверные команды через /sudo (если включено)

### Регистрация администратора

```
/tl admin add 123456789 Steve
```

Где `123456789` — ID пользователя Telegram, а `Steve` — отображаемое имя.

```
/tl admin remove 123456789
```

### Список администраторов

```
/tl admin list
```

Данные администраторов хранятся в `plugins/TelegramLogger/admins.json` и сохраняются между перезапусками.

**Важно:** Администратор должен одновременно:
1. Быть зарегистрированным через `/tl admin add`
2. Быть администратором или создателем группы в Telegram

## Автоматическое восстановление конфигурации

Если `config.yml` повреждён (невалидный YAML, проблемы кодировки и т.д.), плагин автоматически:

1. **Обнаруживает** повреждённый файл при загрузке
2. **Создаёт резервную копию** повреждённого файла как `config.yml.broken.2026-03-15_14-30-00`
3. **Создаёт** новый `config.yml` со всеми значениями по умолчанию
4. **Восстанавливает** любые валидные пары `ключ: значение` из повреждённого файла
5. **Объединяет** восстановленные значения в новую конфигурацию

Это означает, что вы никогда не потеряете свой токен бота, ID чатов или пользовательские сообщения из-за синтаксической ошибки YAML.

## Устранение неполадок

### Бот не отвечает

1. **Проверьте токен бота** — Убедитесь, что он правильно скопирован из BotFather
2. **Бот должен быть администратором группы** — Добавьте бота как администратора в группу Telegram
3. **Проверьте Chat ID** — Используйте RawDataBot для подтверждения правильного ID группы
4. **Режим приватности** — Отправьте `/setprivacy` в BotFather и установите `Disable` для вашего бота
5. **Проверьте `/tl status`** — Показывает, активно ли соединение с ботом

### Сообщения не появляются в Telegram

1. **Проверьте переключатели функций** — Убедитесь, что `enable_join`, `enable_chat` и т.д. установлены в `true`
2. **Проверьте защиту от флуда** — Если одновременно происходит много событий, некоторые могут быть ограничены
3. **Включите режим отладки** — Выполните `/tl debug` и проверьте консоль на ошибки
4. **Проверьте валидность конфигурации** — Выполните `/tl reload` и проверьте сообщения об ошибках

### Сообщения не появляются в Minecraft

1. **Включите функцию** — Установите `send_telegram_messages_to_game: true`
2. **Зарегистрируйтесь как администратор** — Используйте `/tl admin add <ваш_telegram_id> <имя>`
3. **Нужно быть администратором группы** — Ваш аккаунт Telegram должен быть администратором группы И зарегистрированным

### `/sudo` не работает

1. **Включите sudo** — Установите `enable_sudo_command: true` в конфигурации
2. **Проверьте чёрный список** — Команда может быть в `sudo_blacklist`
3. **Зарегистрируйтесь как администратор** — Нужно быть зарегистрированным через `/tl admin add`
4. **Проверьте вывод** — Команда может выполниться, но не произвести видимого вывода

### Дублирование сообщений

Это была известная проблема в v4.x, исправленная в v5.0.0. Если вы всё ещё сталкиваетесь с ней:
1. Убедитесь, что используете v5.0.0 или новее
2. Попробуйте `/tl reload` для перезапуска системы опроса

### Проблемы с темами форума

1. **Thread ID должен совпадать** — Используйте RawDataBot в конкретной теме для получения правильного thread ID
2. **`send_to_thread` должен быть `true`** — Иначе thread_id игнорируется
3. **У бота должны быть права на темы** — Убедитесь, что бот может публиковать в конкретной теме

### Повреждённая конфигурация

Плагин обрабатывает это автоматически в v5.0.0:
1. Повреждённая конфигурация сохраняется как `config.yml.broken.<метка_времени>`
2. Генерируется новая конфигурация
3. Восстанавливаемые значения мигрируются

Также можно исправить проблемы вручную:
1. Проверьте повреждённый файл на синтаксические ошибки YAML
2. Используйте YAML валидатор (например, yamllint.com)
3. Частые проблемы: отсутствующие кавычки, символы табуляции (используйте пробелы), незакрытые строки

## Часто задаваемые вопросы

**В: Работает ли этот плагин с Paper?**
О: Да! TelegramLogger работает с любым сервером на базе Spigot, включая Paper, Purpur и форки.

**В: Могу ли я использовать несколько групп Telegram?**
О: Основные события идут в одну группу/тему. Логирование команд может отправляться в отдельную группу через `command_executes_chat_id`.

**В: Токен бота хранится безопасно?**
О: Токен бота хранится в `config.yml` на вашем сервере. Убедитесь, что файлы сервера надёжно защищены.

**В: Как узнать свой Telegram user ID?**
О: Добавьте [@RawDataBot](https://t.me/rawdatabot) в группу, отправьте сообщение и найдите `"from":{"id": ВАШ_ID}`.

**В: Могут ли обычные игроки использовать команды Telegram?**
О: Нет. Только пользователи, которые одновременно являются зарегистрированными администраторами И администраторами группы Telegram, могут использовать команды.

**В: Есть ли ограничения у `/sudo`?**
О: Да — команды из `sudo_blacklist` блокируются. Вывод обрезается до 3000 символов. Команда выполняется от имени консоли (полные права).

**В: Что происходит, если Telegram недоступен?**
О: Плагин корректно обрабатывает сбои API. События продолжают отслеживаться локально. Сообщения просто не отправляются, пока API не станет доступным снова.

**В: Сколько серверных ресурсов это использует?**
О: Минимально. Вся коммуникация с Telegram асинхронна и выполняется вне основного потока. Система защиты от флуда предотвращает чрезмерные вызовы API.

**В: Можно ли настроить эмодзи в названиях миров?**
О: В настоящее время форматирование названий миров встроено. Пользовательские названия миров форматируются автоматически. Вы можете настроить шаблоны сообщений для добавления собственных эмодзи.
