[SIZE=6][B]What's New in v5.0.0[/B][/SIZE]

[SIZE=5][COLOR=rgb(46, 204, 113)][B]Bug Fixes[/B][/COLOR][/SIZE]
[LIST]
[*][B]Fixed duplicate Telegram messages[/B] — Messages from Telegram were sometimes delivered 2-3 times to Minecraft. Root cause was overlapping long-poll requests. Fixed with an atomic polling guard.
[*][B]Fixed /sudo command crashes[/B] — The sudo command would crash in non-forum groups because it tried to access message_thread_id even when it didn't exist. Now handles missing thread IDs gracefully.
[*][B]Fixed /sudo not showing output[/B] — Previously, /sudo plugins would say "executed successfully" but show no output. Now captures and displays full command output in Telegram.
[*][B]Fixed multi-line message prefix spam[/B] — Multi-line messages from Telegram had the plugin prefix prepended to every line. Now the prefix is applied only once.
[*][B]Fixed HTML entity corruption[/B] — HTML entities like &lt; were being corrupted by the color code stripper. Now HTML entities are preserved correctly through the entire message pipeline.
[*][B]Fixed /reload causing Telegram message flood[/B] — Using /reload from Telegram caused an infinite loop. Reload is now only available in-game.
[*][B]Fixed /sudo crashing with vanilla commands[/B] — Now uses console sender with logger capture for reliable command output.
[*][B]Fixed chat mention plugin garbage in Telegram[/B] — Plugin-injected component tags are now cleaned before forwarding.
[/LIST]

[SIZE=6][B]Installation Guide[/B][/SIZE]

[SIZE=5][B]Step 1: Download and Install[/B][/SIZE]
[LIST=1]
[*]Download the latest [B]TelegramLogger-5.0.1.jar[/B] from [URL='https://github.com/LazizbekDeveloper/TelegramLogger/releases']Releases[/URL]
[*]Place the .jar file in your server's [ICODE]plugins/[/ICODE] folder
[*]Start the server once to generate the default configuration
[*]Stop the server to edit the configuration
[/LIST]

[CODE]
plugins/
  TelegramLogger-5.0.1.jar
  TelegramLogger/
    config.yml          <-- Edit this file
    data.json           <-- Auto-generated statistics
    admins.json         <-- Auto-generated admin registry
[/CODE]

[SIZE=5][B]Step 2: Create a Telegram Bot[/B][/SIZE]
[LIST=1]
[*]Open Telegram and find [URL='https://t.me/botfather']@BotFather[/URL]
[*]Send [ICODE]/newbot[/ICODE] and follow the prompts
[*]Copy the [B]API Token[/B] that BotFather gives you (looks like [ICODE]123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11[/ICODE])
[/LIST]

[B]Important Bot Settings[/B] (send these to BotFather):
[CODE]
/setprivacy    -> Choose your bot -> Disable
/setjoingroups -> Choose your bot -> Enable
[/CODE]

(Optional) Set bot commands for a nice command menu:
[CODE]
/setcommands -> Choose your bot -> Paste this:
status - Show server status
stats - Message statistics
players - Online players
start - Start forwarding
stop - Stop forwarding
debug - Toggle debug mode
tps - Server performance
sudo - Execute server command
help - Show commands
[/CODE]

[SIZE=5][B]Step 3: Get Your Chat ID[/B][/SIZE]
[LIST=1]
[*]Create a Telegram group (or use an existing one)
[*]Add your bot to the group [B]as an administrator[/B]
[*]Add [URL='https://t.me/rawdatabot']@RawDataBot[/URL] to the group temporarily
[*]Look for [ICODE]"chat":{"id": -123456789}[/ICODE] in its response — that number is your [B]Chat ID[/B]
[*]Remove RawDataBot from the group
[/LIST]

[B]For Forum Groups (with topics):[/B]
Send a message in the specific topic where you want notifications. RawDataBot will show a [ICODE]"message_thread_id"[/ICODE] — that's your [B]Thread ID[/B].

[SIZE=5][B]Step 4: Configure the Plugin[/B][/SIZE]

Edit [ICODE]plugins/TelegramLogger/config.yml[/ICODE]:
[CODE=YAML]
# Required: Your bot token from BotFather
bot_token: "123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"

# Required: Your group/channel chat ID
chat_id: "-1001234567890"

# Optional: Thread ID for forum topics
thread_id: "123"
send_to_thread: false

# Enable two-way chat (Telegram messages appear in Minecraft)
send_telegram_messages_to_game: true
[/CODE]

Save the file and run [ICODE]/tl reload[/ICODE] in-game, or restart the server.

[B]Register yourself as an admin[/B] (required for Telegram commands):
[CODE]/tl admin add YOUR_TELEGRAM_USER_ID YourName[/CODE]
To find your Telegram user ID, send a message in the group with RawDataBot and look for [ICODE]"from":{"id": 123456}[/ICODE].


[SIZE=6][B]Configuration Reference[/B][/SIZE]

[SPOILER="Bot Configuration"]
[CODE=YAML]
bot_token: "BOT_TOKEN"                    # Your Telegram bot API token
chat_id: "CHAT_ID"                        # Target group/channel ID
thread_id: "THREAD_ID"                    # Forum topic thread ID
send_to_thread: false                     # Enable forum topic support
send_telegram_messages_to_game: false     # Forward Telegram messages to Minecraft
[/CODE]
[/SPOILER]

[SPOILER="Message Prefix"]
[CODE=YAML]
plugin_prefix: "&6&lTelegramLogger&7 -> &r&a"   # In-game message prefix (supports color codes)
telegram_game_message: "&7[&9TG&7] &c%name% &8>> &f%message%"   # Format for TG->MC messages
[/CODE]
The [ICODE]telegram_game_message[/ICODE] template controls how Telegram messages appear in Minecraft.
Available placeholders: [ICODE]%name%[/ICODE] (sender name), [ICODE]%message%[/ICODE] (message text).
[/SPOILER]

[SPOILER="Prefix Replacements"]
[CODE=YAML]
prefix_replacements:
  "VIP": "[VIP]"
  "ADMIN": "[ADMIN]"
  "Member": "<tg-emoji emoji-id='5368324170671202286'>👤</tg-emoji>"
[/CODE]
Replace ugly in-game rank prefixes with clean text for Telegram messages. The key is the text to find in the player's display name, and the value is what to replace it with. Prefix is fetched from Vault API (works with LuckPerms, PEX, etc.) and falls back to display name extraction if Vault is not installed.

[B]HTML support:[/B] Replacement values support full Telegram HTML including custom emoji ([ICODE]<tg-emoji>[/ICODE]), bold, italic and other tags. Values are sent as raw HTML without escaping. If no replacement matches, the prefix is auto-escaped for safety.
[/SPOILER]

[SPOILER="Server Start/Stop Notifications"]
[CODE=YAML]
enable_server_start_stop: true
server_start_message: "<blockquote>Server started!\nVersion: %version% | Max Players: %max%</blockquote>"
server_stop_message: "<blockquote>Server stopped!</blockquote>"
[/CODE]
Sends a notification to Telegram when the server starts or shuts down.
Placeholders: [ICODE]%version%[/ICODE], [ICODE]%max%[/ICODE].
[/SPOILER]

[SPOILER="Join/Leave Messages"]
[CODE=YAML]
enable_join: true
join_message: "<blockquote>%player% joined the game! (%online%/%max%)</blockquote>"

enable_leave: true
leave_message: "<blockquote>%player% left the game! (%online%/%max%)</blockquote>"
[/CODE]
[/SPOILER]

[SPOILER="First Join (New Player Welcome)"]
[CODE=YAML]
enable_first_join: true
first_join_message: "<blockquote>%player% joined for the first time! Welcome! (%online%/%max%)</blockquote>"
[/CODE]
Triggers only when a player joins the server for the very first time (detected via [ICODE]player.hasPlayedBefore()[/ICODE]).
[/SPOILER]

[SPOILER="Chat Messages"]
[CODE=YAML]
enable_chat: true
chat_message: "<b>%player%</b> -> %message%"
[/CODE]
All in-game chat messages are forwarded to Telegram. User messages are HTML-escaped to prevent injection.
[/SPOILER]

[SPOILER="Advancement Messages"]
[CODE=YAML]
enable_advancement: true
advancement_message: "<blockquote>%player% earned [%advancement%] (%online%/%max%)</blockquote>"
[/CODE]
Recipe unlock advancements are automatically filtered out.
[/SPOILER]

[SPOILER="Death Messages"]
[CODE=YAML]
enable_death: true
death_message: "<blockquote>%death_message% (%online%/%max%)</blockquote>"
[/CODE]
Includes the full Minecraft death message with killer information.
[/SPOILER]

[SPOILER="World Switch Messages"]
[CODE=YAML]
enable_world_switch: true
world_switch_message: "<blockquote>%player% moved: %from_world% -> %to_world% (%online%/%max%)</blockquote>"
[/CODE]

World names are automatically formatted:
[LIST]
[*][ICODE]world[/ICODE] -> Overworld
[*][ICODE]world_nether[/ICODE] -> The Nether
[*][ICODE]world_the_end[/ICODE] -> The End
[*][ICODE]spawn[/ICODE] -> Spawn
[*][ICODE]lobby[/ICODE] -> Lobby
[*]Custom worlds -> Auto-formatted name
[/LIST]
[/SPOILER]

[SPOILER="Chat Filter"]
[CODE=YAML]
enable_chat_filter: true
filtered_words:
  - "badword1"
  - "badword2"
  - "offensive_term"
filtered_message: "<blockquote>%player% used a filtered word. (%online%/%max%)</blockquote>"
[/CODE]

When a filtered word is detected:
[LIST=1]
[*]The original message is [B]not[/B] forwarded to Telegram
[*]A filtered notification is sent instead
[*]In-game admins receive a warning
[/LIST]
[/SPOILER]

[SPOILER="Command Execution Logging"]
[CODE=YAML]
enable_send_command_executes: false
command_execute_message: "<blockquote>%player% -> %command% (%online%/%max%)</blockquote>"
command_executes_chat_id: "CHAT_ID"           # Can be different from main chat
send_command_executes_to_thread: false
command_executes_group_thread_id: "THREAD_ID"
ignored_commands:
  - "/login"
  - "/register"
  - "/l"
  - "/reg"
[/CODE]
Commands in the [ICODE]ignored_commands[/ICODE] list will not be logged. You can use a separate Telegram chat/thread for command logs to keep your main chat clean.
[/SPOILER]

[SPOILER="Telegram Sudo Command"]
[CODE=YAML]
enable_sudo_command: false
sudo_show_output: true
sudo_blacklist:
  - "stop"
  - "restart"
  - "op"
  - "deop"
  - "ban-ip"
[/CODE]

When enabled, registered admins can execute server commands from Telegram:
[CODE]
/sudo list          -> Shows online players
/sudo plugins       -> Lists installed plugins with output
/sudo kill Steve    -> Kills the player Steve
/sudo say Hello!    -> Broadcasts a message
/sudo gamemode creative Steve -> Changes gamemode
[/CODE]

The response shows:
[LIST]
[*]Whether the command succeeded
[*]The actual command that was executed
[*]Which admin ran it
[*]The full command output (if [ICODE]sudo_show_output: true[/ICODE])
[/LIST]
Commands in [ICODE]sudo_blacklist[/ICODE] are blocked from execution for safety.
[/SPOILER]

[SPOILER="Anti-Flood Protection"]
[CODE=YAML]
anti_flood_enabled: true
anti_flood_max_messages: 20     # Maximum messages allowed per window
anti_flood_window_seconds: 10   # Window duration in seconds
[/CODE]

Prevents flooding in situations like:
[LIST]
[*]Mass player join/leave events
[*]Chat spam
[*]Rapid advancement unlocks
[*]Event storms
[/LIST]
Messages that exceed the rate limit are silently dropped.
[/SPOILER]

[SPOILER="Debug Mode"]
[CODE=YAML]
debug_mode: false
[/CODE]
Enable with [ICODE]/tl debug[/ICODE] in-game or [ICODE]/debug[/ICODE] from Telegram. Shows detailed logs for message processing, Telegram API calls, config loading, rate limiting decisions, and admin checks.
[/SPOILER]

[SIZE=6][B]Admin Management[/B][/SIZE]

Admins are Telegram users who can:
[LIST]
[*]Send messages to the Minecraft chat from Telegram
[*]Use Telegram commands (/status, /stats, etc.)
[*]Execute server commands via /sudo (if enabled)
[/LIST]

[B]Registering an Admin:[/B]
[CODE]/tl admin add 123456789 Steve[/CODE]
Where [ICODE]123456789[/ICODE] is the Telegram user ID and [ICODE]Steve[/ICODE] is a display name.

[B]Removing an Admin:[/B]
[CODE]/tl admin remove 123456789[/CODE]

[B]Listing Admins:[/B]
[CODE]/tl admin list[/CODE]

Admin data is stored in [ICODE]plugins/TelegramLogger/admins.json[/ICODE] and persists across restarts.

[COLOR=rgb(231, 76, 60)][B]Important:[/B][/COLOR] An admin must be both:
[LIST=1]
[*]Registered via [ICODE]/tl admin add[/ICODE]
[*]A group administrator or creator in the Telegram group
[/LIST]

[SIZE=6][B]HTML Formatting Guide[/B][/SIZE]

[CODE=HTML]

<i>Italic text</i>
<u>Underlined text</u>
<s>Strikethrough text</s>
<code>Monospace/code text</code>

<blockquote>Quoted text block</blockquote>

[/CODE]

join_message: "<blockquote><b>%player%</b> has joined!\nPlayers: <code>%online%/%max%</code></blockquote>"

[SIZE=6][B]Forum Thread Organization[/B][/SIZE]

Telegram forum groups (supergroups with topics) let you organize different event types into separate threads:

[CODE=YAML]
# Main events go to one thread
send_to_thread: true
thread_id: "12345"

# Command logs go to a different thread (or even a different group)
enable_send_command_executes: true
command_executes_chat_id: "-1001234567890"
send_command_executes_to_thread: true
command_executes_group_thread_id: "67890"
[/CODE]

This keeps your server events organized and easy to navigate.

[SIZE=6][B]Config Auto-Restore System[/B][/SIZE]

If your [ICODE]config.yml[/ICODE] becomes corrupt (invalid YAML, encoding issues, etc.), the plugin automatically:

[LIST=1]
[*][B]Detects[/B] the corrupt file during loading
[*][B]Backs up[/B] the broken file as [ICODE]config.yml.broken.<timestamp>[/ICODE]
[*][B]Creates[/B] a fresh [ICODE]config.yml[/ICODE] with all default values
[*][B]Recovers[/B] any valid key: value pairs from the broken file
[*][B]Merges[/B] the recovered values into the new config
[/LIST]

This means you never lose your bot token, chat IDs, or custom messages due to a YAML syntax error.

The plugin is organized into clean, modular packages:

[CODE]
uz.lazizbekdev.telegramlogger/

    |-- MessageUtils.java            # Color codes, formatting, placeholders
    +-- AntiFloodManager.java        # Rate limiting with sliding window

[SPOILER="Bot Not Responding"]
[LIST=1]

[SPOILER="Messages Not Appearing in Telegram"]
[LIST=1]
[*][B]Check feature toggles[/B] — Make sure enable_join, enable_chat, etc. are true
[*][B]Check anti-flood[/B] — If too many events fire at once, some may be rate-limited
[*][B]Enable debug mode[/B] — Run /tl debug and check console for errors
[*][B]Verify config is valid[/B] — Run /tl reload and check for error messages
[/LIST]
[/SPOILER]

[/LIST]
[/SPOILER]

[SIZE=6][B]FAQ[/B][/SIZE]

[B]Q: Does this plugin work with Paper?[/B]
A: Yes! TelegramLogger works with any Spigot-based server including Paper, Purpur, and forks.

[B]Q: Can I use this with multiple Telegram groups?[/B]
A: The main events go to one group/thread. Command logging can go to a separate group via [ICODE]command_executes_chat_id[/ICODE].

[B]Q: Is the bot token stored securely?[/B]
A: The bot token is stored in config.yml on your server. Make sure your server files are properly secured.

[B]Q: How do I find my Telegram user ID?[/B]
A: Add [URL='https://t.me/rawdatabot']@RawDataBot[/URL] to a group, send a message, and look for [ICODE]"from":{"id": YOUR_ID}[/ICODE].

[B]Q: Can regular players use Telegram commands?[/B]
A: No. Only users who are both registered admins AND Telegram group admins can use commands.

[B]Q: Does /sudo have any limitations?[/B]
A: Yes — commands in sudo_blacklist are blocked. Output is truncated to 3000 characters. The command runs as console (full permissions).

[B]Q: What happens if Telegram is down?[/B]
A: The plugin gracefully handles API failures. Events continue to be tracked locally. Messages are simply not sent until the API is available again.

[B]Q: How much server performance does this use?[/B]
A: Minimal. All Telegram communication is asynchronous and runs off the main thread. The anti-flood system prevents excessive API calls.

[B]Q: Can I customize the emoji in world names?[/B]
A: Currently, world name formatting is built-in. Custom world names are auto-formatted. You can customize the message templates to add your own emoji.
